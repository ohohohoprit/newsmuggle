     7→### Original Repo Analysis
     8→
     9→The original is a Vite + React 19 project (`content-smuggler-react`) themed as a "Content Smuggler" creator toolkit. Key characteristics:
    10→
    11→- **Theme**: Vintage paper / secret agent / smuggler aesthetic. Dark UI with gold (#C09858) and forest-green (#213A28) accents. Cream paper textures (#E6D5B8). Playfair Display + Inter typography. "TOP SECRET" stamps, wax seals, paperclips, "Mission Brief" / "Intel Acquired" terminology.
    12→- **Pages in original**: LandingPage (dark), LoginPage + SignupPage (dark), DashboardPage, AllToolsPage (light paper), GenericToolPage (light paper tool generator with mascot, mission parameters + intel acquired panes), plus PlaceholderPage and a Loader.
    13→- **Routes**: `/`, `/login`, `/signup`, `/dashboard`, `/tools/all`, `/tools/:toolId`, `/studio/*`, `/library/*`, `/account/*`.
    14→- **Stack**: Vite + React 19, Tailwind 4, framer-motion, @phosphor-icons/react, zustand store (filters/sort/favorites), react-router-dom.
    15→- **Data**: `toolsData.js` exports `ALL_TOOLS` array with ~75 tools across 9 categories, each with id/name/desc/category/icon/isPopular/agentTip/uses/color. Includes CATEGORIES list.
    16→
    17→### Port Strategy (Next.js 16 + TS + shadcn/ui)
    18→
    19→Since the user can only see the `/` route in the sandbox, I'm porting this into a **single-page experience with state-driven view switching and modals**:
    20→
    21→1. Landing view (Navbar + Hero + ToolsSection dashboard + AllTools grid + Footer) — all on `/`
    22→2. AuthModal (login/signup toggle)
    23→3. ToolModal (opens when a tool card is clicked — shows the mission parameters + intel acquired workspace)
    24→4. Replaced `react-router-dom` navigation with state-based view switching
    25→5. Replaced `@phosphor-icons/react` with `lucide-react` (already in project)
    26→6. Kept framer-motion, zustand
    27→7. Used shadcn/ui Dialog, Button, Input, Textarea, Select, Sheet, etc. where appropriate
